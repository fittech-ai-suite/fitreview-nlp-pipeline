Methodology

How It Works

End-to-end from raw app store data to a production-ready REST API.

1

Data Collection

Scraped 6,549 reviews across 23 fitness apps (nutrition, cycling, running, yoga, sleep and more) from Google Play. Each review is annotated with 7 aspect flags during EDA.

2

Model Training

Fine-tuned DistilBERT on binary labels (negative / positive). Neutral 3-star reviews dropped — they're ambiguous by design and hurt both precision and recall on the classes that matter.

3

Aspect Analysis

Ran the trained model over all 5,704 non-neutral reviews and aggregated predictions by app, category, and aspect (price, bugs, UI, features, accuracy, ads, support).

Error Analysis

Where the Model Fails

100 / 1,141 test samples misclassified (91.2% correct). The failure modes are consistent and interpretable.

False Positives predicted positive

"A great app in general, but so much stuff in there that I don't need… I don't want meal tips, I don't need all these animations…"
— 1-star review, conf 0.995
Opens with "great app" — the model weights positive framing more than the closing complaint list. Confidence is high because the language is calm, not angry.

False Negatives predicted negative

"Good product, bad customer service. I've had no connectivity issues at all. I find the data to be pretty accurate and useful…"
— 4-star review, conf 0.994
Bug reports written with calm resignation ("since the October update GPS doesn't work") — no emotional language for the model to pick up on.
Usage

Quick Start

Load the model directly from HuggingFace Hub — no local weights needed.

Python 
from src.inference.predictor import SentimentPredictor

model = SentimentPredictor("Anurag717/distilbert-fitness-sentiment-binary")

# single prediction
model.predict("This app completely transformed my training routine")
# → {'label': 'positive', 'confidence': 0.981, 'scores': {'negative': 0.019, 'positive': 0.981}}

# batch inference
model.predict_batch([
    "Crashes every time I try to log a workout",
    "Clean UI, accurate tracking, no complaints after 2 years",
])

# REST API
# uvicorn src.api.app:app --port 8000
# POST /predict  {"text": "..."}
