The 3-class neutral class (3-star reviews) scored F1 = 0.37 — inherently ambiguous text. Switching to binary framing and upgrading to RoBERTa-base with negation-aware augmentation lifted accuracy from 77% to 93.8% (+16.8 points) and fixed the positional-bias failure mode that plagued DistilBERT.

Methodology

How It Works

End-to-end pipeline from raw Google Play data to a production REST API.

1

Data Collection

Scraped 6,549 reviews across 23 fitness apps spanning nutrition, cycling, running, yoga, sleep, and strength training. Each review annotated with 7 aspect flags during EDA.

2

Model Training

Fine-tuned RoBERTa-base with label smoothing, cosine LR schedule, and 303 synthetic negation examples that directly teach contrastive sentence structure — fixing the positional-bias failure of smaller models.

3

Aspect Analysis

Ran the trained model over all 5,704 reviews and aggregated by app, category, and aspect — surfacing which apps and which product dimensions drive negative sentiment.

Error Analysis

Where the Model Fails

100 / 1,141 test samples misclassified — 91.2% correct. Both failure modes are consistent and interpretable.

⚠ False Positive — predicted positive
"A great app in general, but so much stuff in there that I don't need… I don't want meal tips, I don't need all these animations…" — 1★, conf 0.995
Opens with "great app" — the model weights the positive framing more than the closing complaint list. Calm language, not angry.
⚠ False Negative — predicted negative
"Good product, bad customer service. I've had no connectivity issues at all. Data is accurate and useful…" — 4★, conf 0.994
Bug reports written with calm resignation ("since the October update GPS doesn't work") — no emotional language for the model to detect.
Usage

Quick Start

Load directly from HuggingFace Hub — no local weights needed.

python 
from src.inference.predictor import SentimentPredictor

model = SentimentPredictor("Anurag717/distilbert-fitness-sentiment-binary")

# single prediction
model.predict("This app completely transformed my training")
# → {'label': 'positive', 'confidence': 0.981, 'scores': {'negative': 0.019, 'positive': 0.981}}

# batch inference
model.predict_batch([
    "Crashes every time I log a workout",
    "Clean UI, accurate tracking, no complaints after 2 years",
])

# REST API — POST /predict  {"text": "..."}
